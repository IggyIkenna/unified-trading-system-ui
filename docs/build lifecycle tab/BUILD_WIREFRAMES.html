Primary Pipeline Flow (left to right)

• Configure params
• Apply presets
• View dependency tree
• Track versions

Outputs: Feature definitions
→

• Compute features
• Progress by shard
• Trigger jobs / VMs
• Completion heatmap
• Monitor pipeline
Inputs: Feature definitions
Outputs: Computed features
→
3. MODELS
• Model registry
• Pin feature versions
• Train (ETL view)
• Evaluate accuracy
• Compare versions
Inputs: Computed features
Outputs: Trained models
→
4. STRATEGIES
• Configure strategy
• Run signal backtest
• View signal list
• Compare strategies
• Single/Portfolio mode
Inputs: Trained models
Outputs: Signal list
→
5. EXECUTION
• Configure exec algo
• TradingView results
• 5 result sub-tabs
• Compare algos
• Mark candidates
Inputs: Signals
Outputs: Trades / Candidates
Cross-Tab Navigation Hints (clickable links in UI)
• Features expanded row → "View computation jobs" → Feature ETL
• Models expanded row → "Use in Strategy" → Strategies (pre-fills model)
• Strategies signal output → "Send to Execution" → Execution (pre-fills signals)
• Execution results → "Mark as Candidate" → appears in Promote lifecycle tab
• Quant Workspace → "Data Readiness" link → Acquire lifecycle tab
• Quant Workspace → stale features alert → click feature name → Features tab detail
6. QUANT WORKSPACE
Cross-cutting KPI dashboard
• Feature coverage %
• Model health summary
• Best strategies (7d)
• Candidate pipeline
• Data readiness (Acquire)
Reads from: All 5 tabs
Links to: Acquire, Promote
