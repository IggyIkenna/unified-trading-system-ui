# M — Naming Conventions & Consistency Audit

**Date:** 2026-03-28  
**Scope:** `app/`, `components/`, `lib/`, `hooks/` (TypeScript/TSX; excludes `node_modules`, `.next`, generated OpenAPI types except where cited)  
**Previous audit:** First audit

## 1. Current State

- **React components:** Default exports and named exports overwhelmingly use **PascalCase** component identifiers. No `export default function foo` (lowercase) matches were found under `app/`, `components/`, `lib/`, or `hooks/`.
- **Domain & shared UI files:** Most feature and shell code uses **kebab-case** filenames (e.g. `batch-live-rail.tsx`, `trading-vertical-nav.tsx`).
- **Primitives (`components/ui/`):** **kebab-case** filenames align with shadcn/Radix conventions (e.g. `dropdown-menu.tsx`, `alert-dialog.tsx`). Internal component names inside those files remain PascalCase where applicable.
- **Hook functions:** Custom hooks use the **`use` camelCase prefix** on the function name (e.g. `useAuth`, `useIsMobile`, `usePositionsData`). React Query hooks under `hooks/api/` use **kebab-case** filenames (`use-positions.ts`, etc.).
- **Props types:** **`interface …Props`** is the dominant pattern for component props (180+ `interface *Props` matches under `components/` alone).
- **Config / module constants:** Top-level exports in `lib/config/` commonly use **UPPER_SNAKE** or **PascalCase** object keys via structured config objects; local `const` inside functions may use camelCase (e.g. arrays inside helpers).
- **Event handlers:** Inline `onClick={() => …}` is very common (~674 `onClick={` arrow patterns vs ~125 passing a named `handle*` handler); both coexist. Radix/shadcn-style APIs use **`onOpenChange`**, **`open`**, **`disabled`** rather than `isOpen` / `isDisabled` for many primitives—consistent with upstream, not in-house drift.
- **Design tokens (`app/globals.css`):** **kebab-case** custom properties (`--color-primary`, `--spacing-page`, `--text-body`) and Tailwind v4 `@theme` mapping; no BEM class strings required for the main system—utility-first + CSS variables.

## 4. Recommended Fixes

1. **Delete** `components/ui/use-mobile.tsx` after confirming no dynamic import; keep `hooks/use-mobile.ts` as SSOT (**~15 min**).
2. **Rename** `hooks/deployment/use*.ts` to `use-config.ts`, `use-health.ts`, etc., and fix imports (**~1–2 h**).
3. **Batch-rename** `components/ops/deployment/` PascalCase files to kebab-case + update imports (or **document exception** explicitly)—(**~2–4 h** if renaming; **~30 min** if documenting exception).
4. **Add a short “Naming” subsection** to `.cursorrules` or internal docs: hook file kebab-case; component files kebab-case except [if exception]; Radix boolean props allowed (**~30 min**).
5. **Gradually disambiguate** duplicate domain interface names (`Alert`, `ChatMessage`) with prefixes or move shared types to `lib/types/` (**ongoing**, **~0.5–1 d** for high-traffic types).
