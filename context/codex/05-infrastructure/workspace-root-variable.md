# Workspace Root Variable Pattern

## TL;DR

- **User-specific variable**: Each developer sets `UNIFIED_TRADING_WORKSPACE_ROOT` in their shell profile
- **Not committed to git**: Allows sharing workspace configs via PM repo without hardcoding paths
- **Used in scripts**: All workspace-level scripts reference this variable for portability
- **One-time setup**: Set once per machine, works across all workspace operations

---

## Problem

Multi-repo workspace configurations need to reference absolute paths, but developers have different workspace locations:

```bash
# Developer 1
/Users/ikenna/Documents/repos/unified-trading-system-repos/

# Developer 2
/home/harsh/workspace/unified-trading/

# Developer 3
/mnt/c/Users/cosmic/projects/unified-trading-system-repos/
```

Hardcoding paths in scripts breaks portability. Git-tracking absolute paths in configs requires per-developer branches.

---

## Solution: User-Specific Environment Variable

Each developer sets `UNIFIED_TRADING_WORKSPACE_ROOT` once in their shell profile:

```bash
# ~/.zshrc or ~/.bashrc
export UNIFIED_TRADING_WORKSPACE_ROOT="/Users/ikenna/Documents/repos/unified-trading-system-repos"
```

All workspace scripts reference this variable:

```bash
# scripts/workspace/setup-workspace-root.sh
WORKSPACE_ROOT="${UNIFIED_TRADING_WORKSPACE_ROOT}"
cd "${WORKSPACE_ROOT}/unified-trading-system-repos/unified-trading-pm" || exit 1
```

---

## Setup Instructions

### Step 1: Determine Your Workspace Path

```bash
# Navigate to workspace root
cd /path/to/unified-trading-system-repos

# Print absolute path
pwd
```

### Step 2: Add to Shell Profile

**For zsh (macOS default):**

```bash
echo 'export UNIFIED_TRADING_WORKSPACE_ROOT="/Users/your-username/path/to/unified-trading-system-repos"' >> ~/.zshrc
source ~/.zshrc
```

**For bash (Linux default):**

```bash
echo 'export UNIFIED_TRADING_WORKSPACE_ROOT="/home/your-username/path/to/unified-trading-system-repos"' >> ~/.bashrc
source ~/.bashrc
```

**For Windows (Git Bash):**

```bash
echo 'export UNIFIED_TRADING_WORKSPACE_ROOT="/c/Users/your-username/path/to/unified-trading-system-repos"' >> ~/.bashrc
source ~/.bashrc
```

### Step 3: Verify

```bash
echo $UNIFIED_TRADING_WORKSPACE_ROOT
# Should print your workspace root path

cd $UNIFIED_TRADING_WORKSPACE_ROOT
pwd
# Should match your workspace location
```

## Usage in Scripts

### Workspace-Level Scripts

All scripts in `unified-trading-pm/scripts/` use this pattern:

```bash
#!/usr/bin/env bash
set -euo pipefail

# Validate workspace root is set
if [[ -z "${UNIFIED_TRADING_WORKSPACE_ROOT:-}" ]]; then
    echo "ERROR: UNIFIED_TRADING_WORKSPACE_ROOT not set"
    echo "Add to ~/.zshrc: export UNIFIED_TRADING_WORKSPACE_ROOT=\"/path/to/workspace\""
    exit 1
fi

WORKSPACE_ROOT="${UNIFIED_TRADING_WORKSPACE_ROOT}"

# Now safe to reference workspace paths
cd "${WORKSPACE_ROOT}/unified-trading-pm" || exit 1
```

### Per-Repo Scripts

Scripts within individual repos don't need this variable (they use relative paths from repo root):

```bash
#!/usr/bin/env bash
# scripts/quality-gates.sh (runs from repo root)

cd "$(dirname "$0")/.." || exit 1  # Navigate to repo root
source .venv/bin/activate
```

## Workspace Configs That Use This Variable

### 1. Cursor Workspace Settings

`.cursor/workspace-configs/settings.json` references the variable:

```json
{
  "python.analysis.extraPaths": [
    "${env:UNIFIED_TRADING_WORKSPACE_ROOT}/unified-trading-services",
    "${env:UNIFIED_TRADING_WORKSPACE_ROOT}/unified-config-interface"
  ]
}
```

### 2. Ephemeral Rules + Symlink Plans

`.cursor/rules/`, `.cursorrules`, and `AGENTS.md` are **ephemeral copies** from PM. Copied during setup, removed by
cleanup before quickmerge/PR. `.cursor/plans/` remains a symlink into PM.

**Setup:** Per-repo `scripts/setup-workspace.sh` clones PM if needed, then runs
`$PM_DEST/scripts/setup-workspace-from-manifest.sh`, which copies rules, .cursorrules, and AGENTS.md to
`$WORKSPACE_ROOT`.

**Cleanup:** `$WORKSPACE_ROOT/.cleanup-cursor-rules.sh` (generated by setup) removes ephemeral files. Quickmerge runs it
before Stage 5 (Create PR).

### 3. Dependency Checker

`unified-trading-pm/scripts/check-dependency-alignment.sh`:

```bash
WORKSPACE_ROOT="${UNIFIED_TRADING_WORKSPACE_ROOT}"
for repo in "${WORKSPACE_ROOT}"/*/; do
    # Check dependencies
done
```

### 4. Quality Gate Validators

`.cursor/workspace-configs/check-quality-gates.sh`:

```bash
WORKSPACE_ROOT="${UNIFIED_TRADING_WORKSPACE_ROOT}"
cd "${WORKSPACE_ROOT}/instruments-service" && bash scripts/quality-gates.sh --no-fix
```

## Troubleshooting

### Issue: Variable not set after adding to shell profile

**Solution:**

```bash
# Reload shell profile
source ~/.zshrc  # or ~/.bashrc

# Verify
echo $UNIFIED_TRADING_WORKSPACE_ROOT
```

### Issue: Scripts fail with "UNIFIED_TRADING_WORKSPACE_ROOT not set"

**Solution:**

```bash
# Check if variable is set
env | grep UNIFIED_TRADING_WORKSPACE_ROOT

# If empty, add to shell profile (see Step 2)
```

### Issue: Variable set but points to wrong location

**Solution:**

```bash
# Edit shell profile
vim ~/.zshrc  # or ~/.bashrc

# Update the export line
export UNIFIED_TRADING_WORKSPACE_ROOT="/correct/path/to/workspace"

# Reload
source ~/.zshrc
```

### Issue: Works in terminal but not in Cursor

**Solution:**

Cursor may not inherit shell environment variables. Add to Cursor settings:

```json
{
  "terminal.integrated.env.osx": {
    "UNIFIED_TRADING_WORKSPACE_ROOT": "/Users/your-username/path/to/workspace"
  },
  "terminal.integrated.env.linux": {
    "UNIFIED_TRADING_WORKSPACE_ROOT": "/home/your-username/path/to/workspace"
  }
}
```

## Alternative: Per-Developer .env File (NOT RECOMMENDED)

Some teams use a `.env.local` file at workspace root:

```bash
# .env.local (gitignored)
UNIFIED_TRADING_WORKSPACE_ROOT=/Users/ikenna/Documents/repos/unified-trading-system-repos
```

**Why we don't use this:**

- Requires sourcing `.env.local` in every script
- Doesn't work with IDE settings (Cursor/VSCode can't source shell files)
- Adds complexity (shell profile is simpler)

Shell profile is the standard pattern for user-specific environment variables.

## Migration for Existing Developers

If you have existing scripts with hardcoded paths:

### Step 1: Set the variable

```bash
echo 'export UNIFIED_TRADING_WORKSPACE_ROOT="/your/current/workspace/path"' >> ~/.zshrc
source ~/.zshrc
```

### Step 2: Update scripts

Replace hardcoded paths:

```bash
# Before
cd /Users/ikenna/Documents/repos/unified-trading-system-repos/unified-trading-pm

# After
cd "${UNIFIED_TRADING_WORKSPACE_ROOT}/unified-trading-pm"
```

### Step 3: Verify

```bash
# Run updated script
bash unified-trading-pm/scripts/sync-workspace.sh

# Should work without errors
```
