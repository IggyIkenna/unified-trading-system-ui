### SSOT Placement Principle

**What governs the content determines where it lives:**

- Changes because of API/protocol decisions -> `unified-internal-contracts` or `unified-api-contracts`
- Changes because of deployment topology -> `runtime-topology.yaml`
- Changes because of service domain logic -> service-owned config
- Documentation REFERENCES machine-readable SSOTs, never duplicates them
